# 教练手动录入学员功能说明

## 背景

原有流程中，学员须通过教练发送的邀请链接登录后才能与教练绑定。  
新功能支持教练在端内**直接录入学员手机号**，系统自动处理已注册和未注册两种情况。

---

## 核心概念：待激活关系（relation_status = 2）

| relation_status | 含义 |
|---|---|
| 0 | 已解除 |
| 1 | 正常（可约课） |
| **2** | **待激活（教练手动录入未注册手机号，等待学员绑定手机号）** |

待激活关系的特点：
- `student_id` 为 `null`
- `pending_phone` 保存教练录入的手机号
- **不能发起约课**，在学员列表中显示但状态有别于正式关系

---

## 新增接口

### POST `/api/h5/relations/add-by-phone`

教练通过手机号手动录入学员。

**请求头**

```
Authorization: Bearer {token}
```

**请求体**

```json
{
  "phone": "13800138000",       // 必填，手机号
  "student_name": "张三",       // 选填，学员姓名（教练自定义）
  "coach_remark": "备注信息"    // 选填，教练备注
}
```

**响应 - 手机号已注册（直接建立正式关系）**

```json
{
  "success": true,
  "data": {
    "id": 123,
    "student_id": 456,
    "coach_id": 789,
    "pending_phone": null,
    "relation_status": 1,
    "is_pending": false,
    ...
  },
  "message": "学员添加成功"
}
```

**响应 - 手机号未注册（创建待激活关系）**

```json
{
  "success": true,
  "data": {
    "id": 124,
    "student_id": null,
    "coach_id": 789,
    "pending_phone": "13800138000",
    "relation_status": 2,
    "is_pending": true,
    ...
  },
  "message": "学员信息已录入，等待学员绑定手机号后自动激活"
}
```

**错误情况**

| 错误 | 原因 |
|---|---|
| 400 手机号格式不正确 | 不满足 `1[3-9]xxxxxxxxx` 格式 |
| 400 不能将自己添加为学员 | 录入的手机号属于教练本人 |
| 400 该学员已在您的学员列表中 | 正式关系（status=1）已存在 |
| 400 该手机号已在待激活学员列表中 | 同一教练已录入过该未注册手机号 |

## 学员列表接口变更

### GET `/api/h5/relations/my-students`

**变更点：现在同时返回正式学员（status=1）和待激活学员（status=2）**

- 待激活学员的 `student` 字段为 `null`，`pending_phone` 有值
- 支持通过 `keyword` 搜索 `pending_phone`（教练录入的手机号）
- `relation_status` 字段可区分是否已激活

**待激活学员示例数据结构**

```json
{
  "id": 124,
  "student_id": null,
  "coach_id": 789,
  "pending_phone": "13800138000",
  "student_name": "张三",
  "relation_status": 2,
  "student": null,
  "category_lessons": [...],
  ...
}
```

**前端展示建议**

- 正式学员（`relation_status === 1`）：正常显示
- 待激活学员（`relation_status === 2`）：
  - 显示 `student_name`（教练录入的姓名）或手机号展示
  - 标注「待激活」标签
  - **隐藏约课入口**（待激活关系不支持约课）
  - 可提供「复制邀请链接」按钮，引导该学员注册并绑定手机号

---

## 激活时机：学员绑定手机号

当学员通过微信小程序调用 `POST /api/h5/user/decrypt-phone` 绑定手机号时，后端自动：

1. 检查是否存在以该手机号为 `pending_phone` 的待激活关系
2. 若存在：将关系的 `student_id` 填入当前用户 ID，`relation_status` 更新为 `1`，`pending_phone` 清空
3. 若该学员与教练已有正式关系（曾解除后重新激活），则复用已有关系记录

**前端注意事项：**
- 学员绑定手机号成功后，建议刷新「我的教练列表」（`GET /api/h5/relations/my-coaches`），可能出现新的教练关系
- 绑定手机号后可在教练列表中展示「有新教练与你建立了关系」的提示

## 完整流程图

```
【教练端】                   【服务端】                   【学员端】
  │                              │                              │
  │── 录入手机号+姓名 ──────────→│                              │
  │                              │── 查询 users.phone           │
  │                              │                              │
  │         ┌── 已注册 ──────────┤                              │
  │         │                    │── 建立正式关系(status=1)      │
  │← 添加成功(正式关系) ─────────│                              │
  │         │                    │                              │
  │         └── 未注册 ──────────┤                              │
  │                              │── 建立待激活关系(status=2)   │
  │← 添加成功(待激活) ───────────│                              │
  │                              │                              │
  │                              │         ←── 微信登录 ────────│
  │                              │── 创建 users 记录(openid)    │
  │                              │→ 返回 JWT ──────────────────→│
  │                              │                              │
  │                              │    ←── 绑定手机号(decrypt) ──│
  │                              │── users.phone = 手机号       │
  │                              │── 查 pending_phone 待激活关系│
  │                              │── 激活：student_id填入       │
  │                              │         relation_status=1    │
  │                              │         pending_phone=null   │
  │                              │→ 绑定成功 ─────────────────→│
  │                              │                              │
  │                              │         ←── 刷新我的教练列表 │
  │                              │→ 返回教练列表（含新激活教练）→│
```

## 数据库变更（需执行迁移脚本）

执行以下命令完成数据库迁移：

```bash
node scripts/add-pending-student-feature.js
```

变更内容：
- `student_coach_relations.student_id`：由 `NOT NULL` 改为允许 `NULL`
- 新增 `pending_phone VARCHAR(20)` 字段
- 新增 `idx_pending_phone` 索引
- `relation_status` 字段注释更新（新增 `2-待激活` 说明）

---

## 对现有功能的影响

| 功能 | 影响 |
|---|---|
| 邀请链接绑定流程 | 无影响，逻辑不变 |
| 学员约课 | 无影响，待激活关系（status=2）在约课校验时会被拒绝（"学员与教练之间没有有效的师生关系"）|
| 教练约课 | 同上，无影响 |
| 获取我的教练列表 | 无影响，只查 status=1 |
| 获取我的学员列表 | **变更**：现在同时返回 status=1 和 status=2，前端需适配 |
| 解除师生关系 | 兼容：教练可删除待激活关系（通过 `coach_id` 匹配） |
| 更新师生关系 | 兼容：教练可修改待激活关系的 `student_name`、`coach_remark` 等 |
